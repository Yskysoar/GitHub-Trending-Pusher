## 3. Bug修复记录

### Bug 1：TaskService.run_task()方法名不匹配

**问题**：`gui/app.py` 第169行调用了 `self._task_svc.run_task()`，但 `TaskService` 的方法名是 `run_task_now()`，导致 `AttributeError`。

**修复**：将 `self._task_svc.run_task()` 改为 `self._task_svc.run_task_now()`。

**文件**：`gui/app.py`

### Bug 2：CTkMessagebox不存在

**问题**：`gui/pages/settings_page.py` 中使用了 `ctk.CTkMessagebox()`，但 `CTkMessagebox` 不是 customtkinter 内置组件，是独立的第三方包，导致 `AttributeError: module 'customtkinter' has no attribute 'CTkMessagebox'`。

**修复**：在 `gui/components/widgets.py` 中新增自定义 `MessageBox` 组件，替换所有 `CTkMessagebox` 调用。

**影响文件**：
- `gui/components/widgets.py` — 新增 `MessageBox` 类
- `gui/pages/settings_page.py` — 替换4处 `CTkMessagebox` 调用

## 4. 已完成功能详情

### 4.1 定时任务调度集成

- `main.py` 中创建 `TaskService` 实例并调用 `start_scheduler()`
- 使用APScheduler BackgroundScheduler，支持cron和interval两种模式
- `interval_hours≥24`时用cron触发（指定run_time），`<24`时用interval触发
- 应用退出时调用 `stop_scheduler()` 停止调度

### 4.2 开机自启动集成

- 侧边栏CheckBox与`AutoStart.set_autostart()`联动
- 操作Windows注册表`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- 配置变更同步保存到app_config.json

### 4.3 系统托盘（完整实现）

**使用库**：`pystray` + `Pillow`

**功能**：
- 托盘图标（蓝色圆形，PIL动态生成）
- 右键菜单：
  - 显示窗口（默认双击动作）
  - 立即执行
  - 退出
- 通知气泡：
  - 任务完成时弹出"已为您推荐 N 个项目"
  - 任务失败时弹出错误信息
  - 最小化时弹出"应用已最小化到托盘"
- 关闭窗口行为：
  - `minimize_to_tray=True` → 隐藏窗口+托盘通知
  - `minimize_to_tray=False` → 直接退出

**实现方式**：
- 托盘图标在独立守护线程中运行（`threading.Thread(daemon=True)`）
- 通过 `self.after(0, callback)` 从托盘线程安全调用GUI操作
- 退出时调用 `tray_icon.stop()` 停止托盘

### 4.4 首次运行引导集成

- 应用启动时检测`github.token`和`llm.api_key`
- 任一为空时自动弹出引导弹窗
- 引导弹窗支持输入Token/API Key + 创建示例规则
- 示例规则：AI大模型技能（keywords: AI, LLM, Agent, RAG; topics: machine-learning, nlp; language: Python; priority: 8）

### 4.7 边界情况处理

| 模块 | 边界情况 | 处理策略 |
| --- | --- | --- |
| GitHubFetcher | Trending抓取失败 | 降级为Search API查询 |
| RuleMatcher | 无启用规则 | 保留所有仓库，匹配度默认50 |
| RepoEvaluator | 候选仓库为0 | 跳过后续步骤，生成空报告 |
| RepoEvaluator | LLM评估全部失败 | 学习价值得分设为50 |
| RepoEvaluator | 增长数据不可用 | 增长速度得分设为50 |
| LLMSummarizer | JSON解析失败 | 重试一次+默认评分 |
| LLMSummarizer | LLM返回空内容 | 学习价值得分50+降级模板 |
| Scheduler | 任务取消 | 回滚中间数据 |

### 4.8 性能优化

| 优化项 | 实现方式 |
| --- | --- |
| GUI响应性 | 耗时操作在后台线程执行（threading.Thread） |
| API频率控制 | GitHub API保守限制每分钟30次 |
| 超时处理 | httpx连接超时10秒+读取超时30秒，LLM读取超时60秒 |
| 数据库性能 | WAL模式+外键约束+13个索引 |
| 批量操作 | executemany批量插入匹配记录和仓库数据 |
