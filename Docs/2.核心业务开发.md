## 3. 核心实现详情

### 3.1 GitHub数据抓取器（github_fetcher.py）

**类**：`GitHubFetcher`

**核心方法**：

| 方法 | 功能 |
| --- | --- |
| `fetch_trending_repos(languages, since)` | 抓取GitHub Trending页面，失败时降级为Search API |
| `search_repos_by_query(keywords, topics, language, min_stars)` | 按规则条件搜索仓库 |
| `get_readme_content(full_name)` | 获取仓库README（截取前4000字符） |

**降级策略**：
- Trending页面抓取失败 → 自动降级为Search API查询
- 降级查询构造：`stars:>100 + language:xxx + pushed:>N天前`
- 降级来源的仓库 `stars_growth` 设为0，`growth_source` 标记为 `fallback`

**语言名转换**：配置中存储小写（python），Trending URL需首字母大写（Python），通过 `capitalize_language()` 处理特殊大小写（JavaScript、C++、C#等）。

**Token安全处理**：
- 初始化时通过 `sanitize_ascii()` 清洗Token，去除全角空格、中文标点等非ASCII字符
- 避免非ASCII字符在HTTP请求编码时触发 `UnicodeEncodeError`

**API速率限制处理**：
- 检测 `X-RateLimit-Remaining` 响应头
- 认证后保守限制每分钟30次请求
- 403状态码抛出 `GitHubAPIError(1002)`

### 3.2 规则匹配引擎（rule_matcher.py）

**类**：`RuleMatcher`

**核心方法**：`match_rules(repos, rules)`

**三维度匹配度计算**：

```
match_score = keyword_match_ratio × 0.5 + topic_match_ratio × 0.3 + language_match × 0.2
```

| 维度 | 计算方式 | 空条件处理 |
| --- | --- | --- |
| keyword_match_ratio | 关键词在仓库信息中出现的数量 / 关键词总数 | 空列表 → 1.0 |
| topic_match_ratio | topics交集数 / 规则topics总数 | 空列表 → 1.0 |
| language_match | 语言是否一致（0或1） | 空字符串 → 1.0 |

**优先级加成**：
```
最终规则匹配度得分 = min(基础匹配度得分 × (1 + (priority - 5) × 0.02), 100)
```

**匹配判定**：仓库对某条规则的 `match_score > 0` 即视为匹配；对所有启用规则的 `match_score` 均为0则过滤。

**无启用规则时**：保留所有仓库，规则匹配度得分默认50。

### 3.3 综合评估器（repo_evaluator.py）

**类**：`RepoEvaluator`

**核心方法**：`evaluate_repos(matched_repos, top_n, llm_summarizer)`

**评估流程**：

```
Star阈值过滤 → 增长速度计算 → 阶段一初始评分(3维度) → TOP N初筛
→ LLM学习价值评估 → 阶段二最终评分(4维度) → 排名
```

**Star阈值过滤**：
```
有效阈值 = max(全局github.min_stars, 规则级min_stars)
若匹配多条规则且min_stars不同，取其中最大值参与max计算
```

**增长速度得分**：
```
growth_threshold = 有Trending数据的仓库stars_growth的中位数
若中位数为0 → 降级使用均值+1
若过滤后无Trending数据仓库 → 所有仓库增长速度得分设为50
若过滤后有Trending数据的仓库stars_growth全为0 → 所有仓库增长速度得分设为50
增长速度得分 = min(stars_growth / growth_threshold, 2.0) / 2.0 × 100
Search API来源仓库 → 增长速度得分设为50
```

**阶段一：初始评分（三维度，权重归一化）**：
```
W1 = weights.rule_match / (weights.rule_match + weights.star_threshold + weights.growth_speed)
W2 = weights.star_threshold / (weights.rule_match + weights.star_threshold + weights.growth_speed)
W3 = weights.growth_speed / (weights.rule_match + weights.star_threshold + weights.growth_speed)

初始得分 = 规则匹配度得分 × W1 + Star阈值达标度得分 × W2 + 增长速度得分 × W3
```

**阶段二：最终评分（四维度加权）**：
```
最终得分 = 规则匹配度得分 × 0.3 + Star阈值达标度得分 × 0.2
         + 增长速度得分 × 0.2 + 学习价值得分 × 0.3
```

**边界情况处理**（6种场景）：

| 场景 | 处理策略 |
| --- | --- |
| 过滤后候选仓库为0 | 跳过后续步骤，生成空报告 |
| 候选仓库数量少于TOP N | 取全部候选仓库进入LLM评估 |
| 所有仓库增长数据不可用 | 增长速度得分设为50 |
| LLM评估全部失败 | 学习价值得分设为50，仍按四维度最终得分排名 |
| 无启用规则 | 保留所有仓库，规则匹配度得分默认50 |
| 过滤后有Trending数据的仓库stars_growth全为0 | 增长速度得分设为50 |

### 3.4 LLM总结生成器（llm_summarizer.py）

**类**：`LLMSummarizer`

**核心方法**：

| 方法 | 功能 |
| --- | --- |
| `evaluate_learning_value(repos)` | 四维度评估（创新性/代码质量/实用性/社区活跃度） |
| `generate_summary(top_repos)` | 生成Markdown格式总结报告 |
| `save_summary(content, top_repos, ...)` | 保存日志文件+数据库记录 |

**API Key安全处理**：
- 调用LLM前通过 `sanitize_ascii()` 清洗API Key，去除非ASCII字符
- 避免非ASCII字符在HTTP请求编码时触发 `UnicodeEncodeError`

**LLM调用配置**：
- 评估：temperature=0.3, max_tokens=2048, 每批最多5个项目
- 总结：temperature=0.7, max_tokens=4096

**容错处理**（6种场景）：

| 场景 | 处理策略 |
| --- | --- |
| JSON被markdown代码块包裹 | 正则提取```json...```中的内容 |
| JSON解析失败 | 重试一次；仍失败则默认评分5.0 |
| 评分超出0-10范围 | 截断到有效范围 |
| 缺少必要字段 | 缺少维度用5.0填充 |
| 总结生成格式偏差 | 直接使用原始文本 |
| LLM返回空内容 | 学习价值得分设为50；总结用降级模板 |

### 3.5 定时调度器（scheduler.py）

**类**：`Scheduler`、`TaskCallback`

**9步推送任务流水线**：

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | fetch_trending_repos | 获取Trending项目 |
| 1 | search_repos_by_query | 按规则Search API搜索 |
| 1 | 合并去重+截断 | max_repos_per_fetch限制，DB字段JSON反序列化 |
| 2 | match_rules | 规则匹配 |
| 3-6 | evaluate_repos | 综合评估（含LLM评估） |
| 7 | get_readme_content | 获取README |
| 8 | generate_summary | 生成总结报告 |
| 9 | save_summary | 保存日志 |

**取消任务处理**：
- 每步骤间检查取消信号
- 取消后回滚：清空match_records，恢复repositories的eval_score/eval_details/readme_summary为默认值

**定时调度**：
- 使用APScheduler BackgroundScheduler
- interval_hours≥24时用cron触发，<24时用interval触发
- 支持动态修改调度配置
