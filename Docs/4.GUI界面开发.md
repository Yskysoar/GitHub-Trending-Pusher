## 3. 核心实现详情

### 3.1 主窗口（app.py）

**类**：`App(ctk.CTk)`

**窗口配置**：
- 默认尺寸：960×640
- 最小尺寸：800×500
- 主题：跟随系统（可切换浅色/深色）

**布局结构**：
```
┌──────────────┬──────────────────────────┐
│   侧边栏     │       内容区域            │
│  (160px)     │                          │
│              │                          │
│ 🔵 标题      │   当前页面内容            │
│ 🏠 仪表盘    │                          │
│ 📋 推送规则  │                          │
│ 📁 历史记录  │                          │
│ ⚙️ 系统设置  │                          │
│ ─────────── │                          │
│ ▶ 立即执行   │                          │
│ ☐ 开机自启   │                          │
├──────────────┼──────────────────────────┤
│              │       状态栏              │
└──────────────┴──────────────────────────┘
```

**核心功能**：
- 侧边栏导航切换（4个页面）
- 立即执行按钮（执行中禁用+文字变化）
- 开机自启动开关
- 状态栏实时显示任务进度
- 首次运行引导弹窗（检测Token/API Key为空时自动弹出）
- 关闭窗口最小化到托盘（可配置）

**首次运行引导弹窗**：
- 输入GitHub Token和火山方舟API Key
- 可选创建示例推送规则（AI大模型方向）
- 提供"稍后配置"和"开始使用"两个选项

**任务回调处理**：
- `on_progress` → 更新状态栏文字
- `on_complete` → 恢复按钮状态+刷新仪表盘
- `on_error` → 恢复按钮状态+显示错误信息

### 3.2 公共UI组件（widgets.py）

| 组件 | 用途 | 关键属性 |
| --- | --- | --- |
| `StatCard` | 统计卡片 | title + value，`set_value()` 动态更新 |
| `RepoListItem` | 仓库列表项 | rank + full_name + stars_growth + eval_score + language |
| `RuleCard` | 规则卡片 | 规则信息展示 + 编辑/删除按钮，`edit_button`/`delete_button`属性 |

### 3.4 规则管理页面（rules_page.py）

**类**：`RulesPage(ctk.CTkScrollableFrame)`

**功能**：
- 规则列表展示（RuleCard组件）
- 新建规则弹窗（名称*、关键词*、主题、语言、最低Star、优先级滑块、启用开关）
- 编辑规则弹窗（预填现有数据）
- 删除确认弹窗

**规则编辑弹窗字段**：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 规则名称 | Entry | 是 | — |
| 关键词 | Entry | 是 | 逗号分隔 |
| GitHub主题 | Entry | 否 | 逗号分隔 |
| 编程语言 | Entry | 否 | 留空不限 |
| 最低Star数 | Entry | 否 | 0=使用全局配置 |
| 优先级 | Slider | 否 | 1-10，默认5 |
| 启用规则 | CheckBox | 否 | 默认启用 |

### 3.5 历史记录页面（history_page.py）

**类**：`HistoryPage(ctk.CTkScrollableFrame)`

**功能**：
- 历史日志列表（标题+日期+项目数）
- 关键词搜索
- 分页导航（上一页/下一页+页码显示）
- 打开文件（系统默认程序）
- 删除日志（确认弹窗+同时删除磁盘文件）

### 3.6 设置页面（settings_page.py）

**类**：`SettingsPage(ctk.CTkScrollableFrame)`

**5个配置区域**：

| 区域 | 字段 | 特殊功能 |
| --- | --- | --- |
| GitHub配置 | Token(掩码)+测试、抓取间隔、执行时间、启用定时任务、最大抓取数、最低Star | 连接测试按钮 |
| 推送与评估 | 推送数量、4个权重输入 | 权重之和实时校验 |
| 大模型配置 | API Key(掩码)+测试、Base URL、模型名称 | 连接测试按钮 |
| 输出设置 | 日志保存目录 | 浏览按钮（文件对话框） |
| 应用设置 | 主题下拉、最小化到托盘 | 主题切换即时生效 |

**操作按钮**：恢复默认 + 保存设置

**保存逻辑**：
- 敏感字段（Token/API Key）检测掩码值，保留原值不覆盖
- 权重之和校验（必须为1.0，允许0.01误差）
- 保存后主题即时切换
